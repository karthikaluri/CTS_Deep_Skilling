# Design Patterns Summary - Week 1

## Overview
Design patterns are reusable solutions to common problems in software design. They represent best practices and can speed up the development process.

## Categories

### 1. Creational Patterns
Focus on object creation mechanisms:
- **Singleton**: Single instance, global access
- **Factory Method**: Object creation via interface
- **Abstract Factory**: Family of related objects
- **Builder**: Complex object construction
- **Prototype**: Object copying

### 2. Structural Patterns
Deal with object composition and relationships:
- **Adapter**: Convert incompatible interfaces
- **Bridge**: Separate abstraction from implementation
- **Composite**: Treat individual objects and compositions uniformly
- **Decorator**: Add functionality dynamically
- **Facade**: Unified interface to subsystem
- **Flyweight**: Share common state
- **Proxy**: Control access to another object

### 3. Behavioral Patterns
Focus on object collaboration and responsibility distribution:
- **Command**: Encapsulate requests as objects
- **Iterator**: Access elements sequentially
- **Mediator**: Centralize complex communications
- **Memento**: Capture and restore state
- **Observer**: Notify multiple objects of state change
- **State**: Alter behavior based on state
- **Strategy**: Encapsulate interchangeable algorithms
- **Template Method**: Define algorithm skeleton
- **Visitor**: Perform operations on objects
