RECURSIVE ALGORITHM ANALYSIS
============================
1. Fibonacci (Naive):      O(2^n) - Exponential
   - Base case: n <= 1
   - Recursive case: f(n) = f(n-1) + f(n-2)
   - Problem: Multiple redundant calculations

2. Fibonacci (Memoized):   O(n) - Linear
   - Same recursion with caching
   - Eliminates redundant calculations
   - Trade-off: Time for Space

3. Linear Forecasting:     O(n) - Linear
   - Recursive pattern: forecast[i] depends on previous value + trend
   - Real-world: Trend analysis for financial markets

4. Exponential Forecasting: O(n) - Linear
   - Recursive pattern: forecast[i] = value * (1+rate)^i
   - Models compound growth/interest

5. Statistical Recursion: O(n) - Linear
   - Sum: sum(i) = arr[i] + sum(i+1)
   - Variance: accumulated squared differences

RECURSIVE ALGORITHM PATTERNS
============================
Pattern 1: Base Case + Recursive Case
  - Must have termination condition
  - Each call reduces problem size

Pattern 2: Divide and Conquer
  - Split problem into smaller subproblems
  - Solve recursively
  - Combine results

Pattern 3: Backtracking
  - Explore all possibilities
  - Undo choices (backtrack) when needed

Pattern 4: Memoization (Dynamic Programming)
  - Cache results of subproblems
  - Avoid recalculating same values
  - Transforms O(2^n) to O(n)

RECURSION BEST PRACTICES
========================
1. Always define base case(s) to prevent infinite recursion
2. Ensure recursive case makes progress toward base case
3. Consider stack overflow for deep recursion (convert to iteration if needed)
4. Use memoization to optimize repeated calculations
5. Analyze time complexity carefully - recursion can be expensive
6. Profile recursive algorithms before deployment
7. Consider iterative alternatives for better performance
