# Factory Pattern Notes

## Definition
The Factory Pattern is a creational design pattern that provides an interface for creating objects without specifying the exact classes to create.

## Types of Factory Patterns
1. **Simple Factory**: Single factory class creates objects
2. **Factory Method**: Defines interface for object creation in superclass
3. **Abstract Factory**: Creates families of related objects

## Key Concepts
- Encapsulates object creation logic
- Reduces dependency on concrete classes
- Promotes loose coupling
- Makes the system more flexible

- UI element factories

## Advantages & Disadvantages
### Advantages
- Centralized object creation
- Easy to extend with new types
- Reduces code duplication

### Disadvantages
- Can add unnecessary complexity for simple systems
- May create unused objects
